Golang游戏框架Leaf从懵逼到入门

发表于 2019-05-03 | 更新于 2019-08-25 | 分类于 Golang基础 | | 阅读次数：

  这是关于go语言的第一篇总结，大概从去年十一二月份开始断断续续地学习go的语法，年初差不多看完了菜鸟教程，只能说了解它基本的写法（能看懂而已，甚至创建变量都还搞不撑透），它的特性、用法什么的完全还是懵逼的，依然局限在python的语法思维中（虽然python也是半斤八两😂），无法产出（虽然go的语法现在看确实非常简洁），简直蠢哭。

1. 定义并注册消息

定义消息的结构，也就是客户端发送过来的数据结构。服务端定义好数据的结构，比如有什么字段、字段名字叫什么等，客户端按照该结构发送消息。

将定义好的消息(message)结构注册进leaf框架，当服务端收到客户端发送的消息(简称msg)后，会与注册的消息匹配，根据消息的不同，框架调用不同的函数来执行不同的逻辑。就像http的API一样，框架会根据API的不同去匹配不同的执行函数并调用执行。一种消息和一个API类似，比如发送用户注册的消息类似于请求一个用户注册的API接口。

3. 注册消息处理函数

使用go差不多有个把月了，读了一点框架的底层实现，读完后对它又有了一些新的认识。

主要是对于其各模块间的ChanRPC机制以及消息如何逐层转发（长连接监听到消息 ➜ gate模块 ➜ 其他主模块 ➜ 子逻辑）并返回消息有了更深一点的理解，通过执行逻辑来总结下：

总结：Leaf框架的核心主要是：

  • Skeleton(骨架)对象：和它的名字一样，它将模块在结构上统一组织起来，就像一个骨架一样，模块的其他功能及服务都是在它的基础上搭建；
  • agent对象：agent对象主要涉及与用户的行为相关的功能，用户连接服务端、给服务端发送消息、服务端给用户发送消息等都是通过agent对象来实现；
  • ChanRPC：ChanRPC则是不同模块之间信息交互的提供服务，它是通过go的channel数据结构实现的，而它又集成在Skeleton对象中，所以只要不断将消息塞入Skeleton对象的ChanRPC中，模块就能不断从ChanRPC中获取消息，具体是通过Skeleton对象的Run()方法开启一个for循环来监听ChanRPC实现的，因为go的channel是阻塞的，如果channel里面没有消息，会一直等待，而不会一直循环消耗资源。
