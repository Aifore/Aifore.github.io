Linux cron命令定时任务

发表于 2019-08-01 | 更新于 2019-10-21 | 分类于 运维 | | 阅读次数：

简书：Linux之crontab定时任务

1. 格式：

  • minute（m） ： 代表一小时内的第几分，范围 0-59。
  • hour （h） ： 代表一天中的第几小时，范围 0-23。
  • mday （dom） ： 代表一个月中的第几天，范围 1-31。
  • month（mon） ： 代表一年中第几个月，范围 1-12。
  • wday （dow） ： 代表星期几，范围 0-7 (0及7都是星期天)。
  • who ： 要使用什么身份执行该指令，当使用 crontab -e 时，不必加此字段。
  • command（command）：所要执行的指令。

以分为例（其他类推）：

  • *：表示每分钟都要执行

    * * * * * /bin/usershell 每天每分钟执行一次/bin/usershell

  • a-b：表示从第 a 分钟到第 b 分钟这段时间内要执行

    0-12 * * * * /bin/usershell 每天每小时从0到12分钟每分钟执行一次/bin/usershell

  • */n ：表示每 n 分钟个时间间隔执行一次

    * */2 * * * /bin/usershell 每天每2小时执行一次/bin/usershell

  • a, b, c,… ：表示第 a, b, c,… 分钟要执行

    * 1,3,5,7 * * * /bin/usershell 每天每逢1,3,4,7点执行一次/bin/usershell

2. 配置文件

  • 全局配置文件

    • crontab在/etc目录下面存在

    • cron.hourly：每个小时执行一次的job

    • cron.daily：每天执行一次的job

    • cron.weekly：每个星期执行一次的job

    • cron.monthly：每月执行一次的job

    • cron.d：用来存放任何要执行的crontab文件或脚本。系统自动定期需要做的任务，但是又不是按小时/天/星期/月来执行的，那么就放在这个目录下面

    • crontab：

      可在此文件中添加自己需要的cron job，文件一般如下

      01 root run-parts /etc/cron.hourly
      02 4 root run-parts /etc/cron.daily
      22 4       0 root run-parts /etc/cron.weekly
      42 4 1       * root run-parts /etc/cron.monthly

    • cron.deny：用于控制不让哪些用户使用Crontab的功能

  • 用户配置文件

    每个用户都有自己的cron配置文件，通过crontab -e 就可以编辑,一般情况下我们编辑好用户的cron配置文件保存退出后，系统会自动就存放于/var/spool/cron/目录中，文件以用户名命名。

    linux的cron服务是每隔一分钟去读取一次/var/spool/cron、/etc/crontab、/etc/cron.d下面所有的内容。

3. 命令

  • 检测service路径

    which service

  • 执行相关 cron命令

  • 命令格式

    crontab [ -u user ] 文件
    crontab [ -u user ] { -l | -r | -e }

    • -u：指定某一用户

    • -e：执行vi来设定用户(当前用户或指定用户)时程表

    • -r：删除用户时程表
    • -l：列出用户时程表

  • 添加定时任务推荐方式：

    • 编辑任务脚本【分目录存放】【ex：backup.sh】
    • 编辑定时文件【命名规则：backup.cron】
    • crontab命令添加到系统crontab backup.cron
    • 查看crontab列表 crontab -l

  • @reboot

    为了达到在开机后运行，且只运行一次的目的。除了这个，也无法通过前五段的设置来实现。

    @annually也是这个功能。

    至于@yearly、@monthly等等其实都可以用上面的五段来设置。
