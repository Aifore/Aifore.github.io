1. datetime模块

  1. 其实可以不用将时间格式化成字符串，一是不管时间被转换成什么格式的字符串，最后django在页面解析的时候都会解析成2018年7月7日 00:03这种格式；二是django可以直接解析[datetime.datetime]时间对象。

  2. django在数据库中储存的时间形式为UTC标准时间；

  3. 东八区比UTC标准时间早8小时，django储存到数据库时会自动转换为少8小时的UTC标准时间，取用和页面解析时又将UTC标准时间自动加上8个小时…坑！

    • 在mysql中执行 SET GLOBAL time_zone=’+08:00’；

    • django默认是有运用时区的，需要在设置中把USE_TZ=False；
